## 💰 盈利原理图解

### 正常市场状态

```
┌─────────────────────────────────────┐
│   YES 价格 + NO 价格 ≈ 1.00          │
│                                     │
│   YES: 0.50  │  NO: 0.50  │ Sum: 1.00│
│   YES: 0.60  │  NO: 0.40  │ Sum: 1.00│
│   YES: 0.70  │  NO: 0.30  │ Sum: 1.00│
│                                     │
│   ❌ 无套利机会                     │
└─────────────────────────────────────┘
```

### 剧烈波动状态（套利机会）

```
┌─────────────────────────────────────┐
│   YES 价格快速下跌                   │
│                                     │
│   T+0s:  YES: 0.50  │  NO: 0.50     │
│          ↓                            │
│   T+3s:  YES: 0.40  │  NO: 0.52     │
│          ↓         ↑                 │
│       下跌20%    上涨4%              │
│                                     │
│   Sum: 0.40 + 0.52 = 0.92 < 0.93    │
│                                     │
│   ✅ 套利机会出现！                  │
└─────────────────────────────────────┘
```

### 执行对冲

```
┌─────────────────────────────────────┐
│   Leg1: 买入 YES @ 0.40             │
│   ├─ 花费: 100 USDC                 │
```

## 📈 价格监控逻辑

```
┌──────────────────────────────────────────────────────────┐
│                    价格监控流程                           │
└──────────────────────────────────────────────────────────┘

每秒执行:
    │
    ├─ 获取市场价格 (YES, NO)
    │   ├─ WebSocket 实时推送
    │   └─ REST API 轮询备用
    │
    ├─ 记录价格历史 (保留最近100个点)
    │
    └─ 检测快速下跌
        │
        ├─ 查找 3秒前的价格点
        │
        ├─ 计算下跌幅度:
        │   dropPct = (oldPrice - newPrice) / oldPrice
        │
        └─ 判断是否触发:
            │
            ├─ dropPct >= 15% ?
            │   ├─ 是 → 发送信号
            │   └─ 否 → 继续监控
            │
            └─ 发送信号内容:
                ├─ side: YES 或 NO
                ├─ dropPct: 下跌百分比
                ├─ fromPrice: 原价格
                ├─ toPrice: 新价格
                └─ duration: 持续时间
```

## 🎯 策略参数影响

### 对冲阈值 (Sum Target)

```
Sum Target = 0.90 (保守)
├─ 触发难度: 高 (需要更大价差)
├─ 单笔利润: 高 (~10 USDC)
└─ 交易频率: 低

Sum Target = 0.93 (平衡) ⭐ 当前配置
├─ 触发难度: 中等
├─ 单笔利润: 中等 (~7 USDC)
└─ 交易频率: 中等

Sum Target = 0.95 (激进)
├─ 触发难度: 低 (小价差即可)
├─ 单笔利润: 低 (~5 USDC)
└─ 交易频率: 高
```

### 下跌阈值 (Move %)

```
Move % = 10% (激进)
├─ 触发机会: 多
├─ 风险: 价格可能反弹
└─ 假信号率: 高

Move % = 15% (平衡) ⭐ 当前配置
├─ 触发机会: 适中
├─ 风险: 中等
└─ 假信号率: 中等

Move % = 20% (保守)
├─ 触发机会: 少
├─ 风险: 低
└─ 假信号率: 低
```

## ⚠️ 风险场景

### 场景1: Leg1执行后无对冲机会

```
Leg1 买入 YES @ 0.40
└─ 等待 NO 价格满足对冲条件

NO 价格始终 > 0.53
├─ Sum > 0.93 (不满足对冲)
└─ 持续等待...

周期结束 (T+15分钟)
├─ 无法执行 Leg2
├─ 持有 YES 代币
└─ 等待结算结果

结果不确定性:
├─ YES 胜出: 利润 +60 USDC ✅
└─ NO 胜出: 亏损 -40 USDC ❌
```

**应对措施**:
- 设置合理的 Sum Target
- 优先选择高流动性市场
- 监控网络延迟

### 场景2: 市场流动性不足

```
查找市场
└─ 筛选条件:
    ├─ Spread < 5%
    ├─ Status = ACTIVE
    └─ 包含 "Bitcoin 15 minute"

无合适市场
└─ 本轮跳过，等待下一周期
```

**应对措施**:
- 扩大市场筛选范围
- 调整流动性阈值
- 监控市场列表更新

## 📊 性能指标

```
┌─────────────────────────────────────┐
│         关键性能指标                 │
└─────────────────────────────────────┘

成功率 = 成功轮次 / 总轮次
├─ 目标: > 30%
└─ 说明: 包含Leg1+Leg2完整执行

平均利润 = 总利润 / 成功轮次
├─ 目标: > 5 USDC
└─ 说明: 扣除手续费后

最大回撤 = 单轮最大亏损
├─ 目标: < 50 USDC
└─ 说明: Leg1失败时的最大损失

夏普比率 = (平均收益 - 无风险利率) / 收益波动
├─ 目标: > 1.5
└─ 说明: 风险调整后收益

盈亏比 = 平均利润 / 平均亏损
├─ 目标: > 2.0
└─ 说明: 每承担1单位风险可获得2单位收益
```
